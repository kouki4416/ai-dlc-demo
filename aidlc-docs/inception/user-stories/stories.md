## EP-AUTH. 認証 & ロール管理

### US-001 [Must]: アカウント登録
**As a** Submitter, **I want** メールアドレスとパスワードでアカウントを登録したい, **so that** ポータルを利用開始できる。
- **Given** 未登録のメールアドレスを入力 **When** 登録ボタンを押下 **Then** アカウントが作成され、ログイン画面へ遷移する
- **Given** 既存メールアドレスを入力 **When** 登録ボタンを押下 **Then** "既に登録済み" のエラーが表示される
- **Given** 弱いパスワードを入力 **When** 登録ボタンを押下 **Then** "パスワード要件未達" のエラーが表示される

### US-002 [Must]: ログイン
**As a** すべてのユーザー, **I want** メールとパスワードでログインしたい, **so that** ロールに応じた機能を使える。
- **Given** 正しい認証情報 **When** ログイン **Then** 自分のロールに応じたトップページへ遷移する
- **Given** 誤った認証情報 **When** ログイン **Then** "認証失敗" エラーが表示される
- **Given** 5回連続失敗 **When** さらにログイン試行 **Then** 一時的にロックされる旨のメッセージが表示される

### US-003 [Should]: パスワードリセット
**As a** すべてのユーザー, **I want** パスワードを忘れた時にリセットしたい, **so that** アカウントへの再アクセスができる。
- **Given** 登録済みメールアドレスを入力 **When** リセット要求 **Then** リセットリンクが（仮想的に）発行される ※ メール送信なし、画面表示のみ
- **Given** 有効なリセットトークン **When** 新パスワードを設定 **Then** 新パスワードでログインできる

### US-004 [Must]: ログアウト
**As a** すべてのユーザー, **I want** ログアウトしたい, **so that** セッションを安全に終了できる。
- **Given** ログイン中 **When** ログアウトボタンを押下 **Then** セッションが破棄され、ログイン画面へ戻る

### US-005 [Must]: パネルメンバー任命
**As an** Administrator, **I want** ユーザーをパネルメンバーに任命したい, **so that** 評価チームを編成できる。
- **Given** 一般ユーザーリストを表示 **When** 任命ボタンを押下 **Then** そのユーザーのロールが Panel に変更される
- **Given** すでに Panel ロール **When** 任命を試みる **Then** "既に Panel" の警告が出る

### US-006 [Should]: パネルメンバー解除
**As an** Administrator, **I want** パネルメンバーを解除したい, **so that** チーム構成を見直せる。
- **Given** Panel ロールのユーザー **When** 解除ボタンを押下 **Then** ロールが Submitter に戻る
- **Given** 評価期間中の Panel **When** 解除を試みる **Then** "サイクル終了後に解除を" の警告が出る

### US-007 [Must]: ロール別ナビゲーション
**As a** すべてのユーザー, **I want** 自分のロールに応じたメニューを見たい, **so that** 関係ない機能で迷わない。
- **Given** Submitter としてログイン **When** トップページ表示 **Then** 投稿/ダッシュボード/殿堂のみ表示される
- **Given** Panel としてログイン **When** トップページ表示 **Then** 評価ダッシュボードが追加表示される
- **Given** Admin としてログイン **When** トップページ表示 **Then** 管理メニューが追加表示される

## EP-SUBMIT. アイデア投稿

### US-008 [Must]: 新規アイデア作成
**As a** Submitter, **I want** タイトル・概要・課題・提案の4項目を入力して投稿したい, **so that** 自分のアイデアを発信できる。
- **Given** 4項目すべて入力済み **When** 公開ボタンを押下 **Then** 投稿が確定し、自分のドラフト一覧から外れて公開済み一覧に移動する
- **Given** 必須項目が空 **When** 公開ボタンを押下 **Then** 該当フィールドにエラー表示が出る

### US-009 [Must]: ドラフト自動保存
**As a** Submitter, **I want** 入力中に自動でドラフトが保存されてほしい, **so that** ブラウザクラッシュや誤操作でも内容を失わない。
- **Given** フォーム入力中 **When** 5〜10秒間入力が止まる **Then** ドラフトが自動保存され、UI に "保存済み" が表示される
- **Given** 保存失敗（ネットワーク断） **When** 自動保存 **Then** エラー通知 + 自動リトライ

### US-010 [Must]: ドラフト手動保存
**As a** Submitter, **I want** 手動保存ボタンを押してドラフトを保存したい, **so that** 任意のタイミングで明示的に保存できる。
- **Given** フォーム入力中 **When** "ドラフト保存" を押下 **Then** 即座にドラフト保存され、成功メッセージが表示される

### US-011 [Should]: 画像添付
**As a** Submitter, **I want** PNG/JPG 画像を最大3枚添付したい, **so that** 視覚的にアイデアを補足できる。
- **Given** 5MB 以下の PNG/JPG **When** 添付 **Then** プレビューが表示され、保存される
- **Given** 5MB 超 または PNG/JPG 以外 **When** 添付を試みる **Then** "ファイル形式/サイズエラー" が表示される
- **Given** 既に3枚添付済み **When** 4枚目の添付を試みる **Then** "上限3枚" エラーが表示される

### US-012 [Must]: ドラフト一覧
**As a** Submitter, **I want** 自分の保存済みドラフトを一覧で見たい, **so that** 続きから編集できる。
- **Given** ドラフトが複数 **When** ドラフト一覧を開く **Then** タイトル・更新日時順で表示される
- **Given** ドラフト0件 **When** 一覧を開く **Then** "ドラフトなし、新規作成しましょう" のエンプティステートが表示される

### US-013 [Must]: ドラフトを公開
**As a** Submitter, **I want** ドラフトを公開（投稿確定）したい, **so that** 評価対象になる。
- **Given** 必須4項目入力済みのドラフト **When** "公開" を押下 **Then** 投稿確定の確認ダイアログが出て、確認後ステータスが Published に変わる
- **Given** 必須項目が空のドラフト **When** "公開" を押下 **Then** 公開できず、エラーが表示される
- **Given** 公開後 **Then** 投稿者本人以外には匿名で表示される

### US-014 [Must]: 自分の投稿一覧
**As a** Submitter, **I want** 自分の公開済み投稿を一覧で見たい, **so that** スコアと順位を追跡できる。
- **Given** 公開済み投稿が複数 **When** "自分の投稿" を開く **Then** タイトル・公開日・現在スコア（評価中なら "評価中"）が表示される

### US-015 [Must]: 投稿後編集不可
**As a** Submitter, **I want** 投稿後はアイデアを編集できないことを理解したい, **so that** 評価の整合性が保たれる。
- **Given** 公開済み投稿 **When** 編集ボタンを探す **Then** 編集 UI は表示されない
- **Given** 公開済み投稿 **When** API で直接編集を試みる **Then** 403 エラーが返る

### US-016 [Must]: 入力バリデーション
**As a** Submitter, **I want** タイトル255文字、概要500文字の制限を入力時に確認したい, **so that** 投稿前に修正できる。
- **Given** タイトル255文字超 **When** 入力 **Then** カウンタが赤くなり、保存・公開がブロックされる
- **Given** 概要500文字超 **When** 入力 **Then** 同様

## EP-EVAL. 評価 & スコアリング

### US-017 [Must]: 評価対象アイデア一覧
**As a** Panel Member, **I want** 自分が評価すべきアイデア一覧を見たい, **so that** 評価キューを管理できる。
- **Given** 公開済みアイデアが存在 **When** 評価ダッシュボードを開く **Then** 全公開アイデアがリスト表示され、自分の評価ステータス（未着手/進行中/確定）が各行に表示される

### US-018 [Must]: アイデア詳細閲覧（匿名）
**As a** Panel Member, **I want** アイデアの内容を匿名状態で読みたい, **so that** バイアスなく評価できる。
- **Given** 公開済みアイデア **When** 詳細ページを開く **Then** タイトル・概要・課題・提案・添付画像が表示され、投稿者氏名は表示されない
- **Given** 自身の投稿（自分が Panel 兼任の場合） **When** 詳細を開く **Then** "自分の投稿は評価不可" として評価 UI が無効化される

### US-019 [Must]: 3軸スコア入力
**As a** Panel Member, **I want** Feasibility / Impact / Innovation の3軸を1〜5で評価したい, **so that** 多面的な評価ができる。
- **Given** アイデア詳細ページ **When** 各軸でスコア（1〜5）を選択 **Then** スコアが下書きとして保存される
- **Given** 1〜5 以外を入力（API 経由） **When** 送信 **Then** バリデーションエラー

### US-020 [Should]: コメント入力
**As a** Panel Member, **I want** 任意でスコア理由のコメントを残したい, **so that** 後で振り返りや投稿者フィードバックに活用できる。
- **Given** スコア入力中 **When** コメント欄に記入 **Then** スコアと一緒に保存される
- **Given** コメント空欄 **When** 評価確定 **Then** 警告なしで確定できる（コメントは任意）

### US-021 [Must]: スコア修正
**As a** Panel Member, **I want** 確定前に自分のスコアを修正したい, **so that** 内省を反映できる。
- **Given** 下書き状態のスコア **When** 値を変更 **Then** 上書き保存される
- **Given** 確定済みのスコア **When** 修正を試みる **Then** "確定済みは修正不可" と表示される

### US-022 [Must]: 評価確定
**As a** Panel Member, **I want** 自分のスコアを確定して提出したい, **so that** 集計に反映できる。
- **Given** 全3軸入力済み **When** "評価確定" を押下 **Then** 確認ダイアログが出て、確認後ステータスが "確定" になる
- **Given** いずれかの軸が空 **When** 確定を試みる **Then** "全軸入力必須" エラーが表示される

### US-023 [Should]: 同時並行評価ダッシュボード
**As a** Panel Member, **I want** 複数アイデアを同時並行で評価するダッシュボードを使いたい, **so that** 評価効率が上がる。
- **Given** 複数アイデアが評価対象 **When** ダッシュボードを開く **Then** タブまたはサイドリストで切り替えながら評価できる
- **Given** あるタブで未保存変更 **When** 別タブへ切替 **Then** 自動保存されてから切替

### US-024 [Must]: 他パネルスコア非公開
**As a** Panel Member, **I want** 評価期間中、他パネルのスコアやコメントを見えないようにしたい, **so that** 独立性が保たれる。
- **Given** 評価期間中 **When** アイデア詳細を開く **Then** 自分のスコアのみ表示、他パネルのスコア・コメントは非表示
- **Given** 評価期間終了後 **When** 同じアイデア詳細を開く **Then** （Admin と投稿者本人のみ）詳細スコアを閲覧可能

## EP-DASH. ダッシュボード & 分析

### US-025 [Must]: 全アイデアリーダーボード
**As a** すべてのユーザー, **I want** 公開アイデアを総合スコア順で見たい, **so that** 注目アイデアを把握できる。
- **Given** 公開アイデアが存在 **When** リーダーボードを開く **Then** 総合スコア降順でアイデアが一覧表示される
- **Given** 評価中（一部パネルのみ確定） **Then** "N/M完了" のステータスとともに暫定スコアが表示される

### US-026 [Should]: 軸別ランキング
**As a** すべてのユーザー, **I want** Feasibility / Impact / Innovation 各軸の上位ランキングを見たい, **so that** 観点別に注目アイデアが分かる。
- **Given** 公開アイデアが存在 **When** 軸タブを切替 **Then** 選択軸の平均スコア降順で再ソートされる

### US-027 [Must]: アイデア詳細ページ
**As a** すべてのユーザー, **I want** 個別アイデア詳細でスコア分布をビジュアルで見たい, **so that** 評価結果を直感的に把握できる。
- **Given** アイデア詳細を開く **When** 評価が確定済み **Then** 3軸の平均スコアがレーダーチャートまたは棒グラフで表示される
- **Given** 評価中 **Then** "評価中" ラベルと暫定スコアが表示される

### US-028 [Should]: 投稿状況/評価進捗表示
**As a** すべてのユーザー, **I want** 全体の投稿数・評価進捗を見たい, **so that** サイクルの進行状況が分かる。
- **Given** ダッシュボードを開く **When** サマリエリアを表示 **Then** 総投稿数、評価対象数、パネル別進捗率が表示される

### US-029 [Must]: 準リアルタイム更新
**As a** すべてのユーザー, **I want** リーダーボードが数秒〜数十秒で更新されてほしい, **so that** 最新状況を追える。
- **Given** リーダーボード表示中 **When** 別パネルが評価確定 **Then** 5〜30秒以内に表示が更新される（SSE / 短間隔ポーリング）

### US-030 [Should]: ディメンション別比較
**As a** すべてのユーザー, **I want** 3軸ごとに上位アイデアとスコア分布を比較したい, **so that** 観点別の強みが見える。
- **Given** 比較ビューを開く **When** 3軸並列表示 **Then** 各軸の上位N件と分布チャートが並んで表示される

## EP-REC. 表彰 & 殿堂

### US-031 [Must]: 評価サイクル終了処理（自動集計）
**As an** Administrator, **I want** サイクル終了時に自動で総合スコアを確定したい, **so that** 表彰準備が整う。
- **Given** サイクル期間が終了 **When** Admin が "サイクル終了" を実行 **Then** 全アイデアの最終スコアが確定し、暫定状態のスコアも凍結される
- **Given** 一部パネルが未確定 **When** 終了処理 **Then** 警告ダイアログが表示され、Admin が強制終了か延長かを選択

### US-032 [Must]: 上位3アイデア決定
**As a** システム, **I want** 総合スコア順に上位3アイデアを自動決定したい, **so that** 表彰対象が明確になる。
- **Given** サイクル終了 **When** 集計確定 **Then** 総合スコアの降順で上位3アイデアが Hall of Fame 候補として識別される
- **Given** 同点 **When** 集計 **Then** 同点処理ルール（例: Innovation スコアでタイブレーク）が適用される ※ 詳細ルールは Construction フェーズで決定

### US-033 [Must]: 上位3名の氏名自動公開
**As a** システム, **I want** 上位3アイデアのみ投稿者氏名を自動で公開したい, **so that** 表彰が成立する。
- **Given** サイクル終了 + 上位3確定 **When** 表彰公開 **Then** 上位3アイデアの投稿者氏名のみリーダーボード/詳細/殿堂に表示される
- **Given** 4位以下のアイデア **Then** 匿名状態が維持される

### US-034 [Must]: 表彰殿堂ページ
**As a** すべてのユーザー, **I want** Hall of Fame ページで歴代受賞アイデアを見たい, **so that** イノベーション文化が可視化される。
- **Given** 殿堂ページを開く **When** 表示 **Then** 直近サイクルの上位3（タイトル・概要・スコア・投稿者氏名）が表示される
- **Given** 過去のサイクルがある **When** サイクル切替 **Then** 各サイクルの上位3が遡って閲覧できる

### US-035 [Should]: 殿堂履歴
**As a** すべてのユーザー, **I want** 過去すべての受賞履歴を一覧で見たい, **so that** 累積的な貢献が把握できる。
- **Given** 殿堂履歴ページを開く **When** 表示 **Then** サイクル日付降順で全受賞リストが表示される

## EP-ADMIN. 管理機能

### US-036 [Must]: 評価サイクル作成
**As an** Administrator, **I want** 新しい評価サイクルを作成したい, **so that** 評価期間を区切って運営できる。
- **Given** Admin としてログイン **When** "新規サイクル作成" を押下、開始/終了日を入力 **Then** サイクルが作成され、対象期間内の投稿が評価対象に含まれる
- **Given** 既存のアクティブサイクル **When** 重複作成を試みる **Then** "既存サイクルがアクティブ" エラー

### US-037 [Must]: 評価サイクル終了
**As an** Administrator, **I want** サイクルを終了させたい, **so that** 表彰確定に進める。
- US-031 と連動。終了処理は Admin の明示的な操作で発火する。

### US-038 [Must]: パネルメンバー一覧管理
**As an** Administrator, **I want** 現在のパネルメンバー一覧を見て任命/解除したい, **so that** チーム構成を維持できる。
- **Given** Admin ロール **When** パネル管理ページを開く **Then** 現 Panel メンバー一覧と、任命候補（一般ユーザー）リストが表示される
- US-005 / US-006 と連動。

### US-039 [Should]: 不適切投稿削除
**As an** Administrator, **I want** 不適切なアイデアを削除したい, **so that** ポータルの品質を保てる。
- **Given** 公開済みアイデア **When** Admin が "削除" を押下 + 理由入力 **Then** ステータスが Removed に変わり、リーダーボードから除外。監査ログに記録される
- **Given** 既に評価済みのアイデア **When** 削除 **Then** 評価データは保持しつつ、表示のみ非表示

### US-040 [Could]: アーカイブ済みデータ閲覧
**As an** Administrator, **I want** 1年経過してアーカイブされたアイデアを閲覧したい, **so that** 過去資産を参照できる。
- **Given** アーカイブ済みアイデアが存在 **When** Admin がアーカイブ閲覧ページを開く **Then** アーカイブ一覧が表示される（一般ユーザーには非表示）

### US-041 [Should]: 管理者向けメトリクス閲覧
**As an** Administrator, **I want** 総投稿数・評価進捗率・各軸の平均スコア分布を見たい, **so that** 運営状況を把握できる。
- **Given** Admin ダッシュボード **When** メトリクスタブを開く **Then** 総投稿数、期間別投稿数、パネル別評価完了率、各軸の平均/分布が表示される

## INVEST 自己レビュー

| 項目 | 評価 | 備考 |
|---|---|---|
